# 第二章：英雄觉醒 (变量与类型)

> 进入魔塔后，你需要重塑肉身。在这一层，你将定义自己的“英雄属性”。这些属性将决定你在塔内的生存能力。

## 本章目标

*   区分可变与不可变变量的用途。
*   熟悉常见基础类型及其使用场景。
*   理解类型安全对战斗系统的保护作用。

## 1. 塑造肉身 (变量定义)

魔塔法则规定：有些属性是与生俱来的（不可变），有些则是随着战斗变化的（可变）。

```cangjie
main() {
    // 灵魂烙印（不可变）
    let heroName: String = "Cangjie Hero"
    let heroClass = "Mage" // 自动感知职业为 String

    // 肉体凡胎（可变）
    var level: Int64 = 1          // 初始等级
    var hp: Float64 = 100.0       // 生命值
    var mp: Float64 = 50.0        // 法力值
    var isAlive: Bool = true      // 存活状态

    println("=== 英雄诞生 ===")
    println("姓名: ${heroName} | 职业: ${heroClass}")
    println("状态: HP ${hp} | MP ${mp} | Lv ${level}")
}
```

## 2. 属性法则 (基础类型)

*   **Int64**: 你的等级、金币。整数是世界的基石。
*   **Float64**: 你的生命与法力。世界不是非黑即白的，它有精度。
*   **Bool**: 生与死 (`true`/`false`)，这是二元法则。
*   **String**: 你的名字和符文。

## 3. 严酷的法则 (类型安全)

魔塔不允许混沌。你不能将“生命值”（数字）变成“名字”（文字）。

```cangjie
// hp = "一百" // ❌ 禁忌：类型不匹配！魔塔拒绝了你的请求。
```
只有遵守法则，才能在塔中存活。

## 工程化提示

*   默认使用不可变变量，能减少状态被意外修改的风险。
*   生命值、法力值等数据要统一单位和精度，避免“混用”。
*   角色属性建议集中管理，避免散落在多处逻辑中。
