# 第一章：系统启动 (Hello World)

> 欢迎加入“未来家”智能家居研发团队。你的第一个任务是编写控制中枢的启动程序。这不仅是一行代码，更是整个智能生态的起点，也是后续日志、监控、故障定位的入口。

## 本章目标

*   了解系统启动程序的最小结构。
*   熟悉日志输出与启动流程的基本习惯。
*   建立“代码编译-部署-运行”的工程思维。

## 1. 启动信号

任何系统都需要一个入口。在仓颉中，`main()` 函数就是这个开关。

```cangjie
// file: smart_home_boot.cj
main() {
    println(">>> 智能家居控制中枢正在启动...")
    println(">>> 系统自检完成。准备就绪。")
}
```

*   **现实意义**：`println` 是最基础的日志输出工具。在实际开发中，清晰的日志是系统运维的第一道防线。

## 2. 编译部署

将你的源码编译为可执行的固件。

```bash
cjc smart_home_boot.cj -o boot_loader
./boot_loader
```

当终端打印出启动信息时，意味着你的代码已经成功接管了硬件的控制权。

## 语言特性与应用解读

*   `main()` 是顶层函数入口，默认返回 `Unit`，无需额外声明返回类型。
*   `println` 来自标准库，适合在启动阶段输出关键事件与阶段标记。
*   字符串模板 `${}` 可以嵌入表达式，让启动日志保持清晰可读。

在真实系统中，入口函数只负责编排步骤，具体的自检、配置加载建议拆分成独立函数，便于测试与复用。

## 工程化提示

*   启动日志建议包含模块名与版本号，便于部署排查。
*   `main` 中只做启动编排，避免把业务逻辑塞入入口。
*   真实固件部署还需要配置管理与失败回滚机制。
