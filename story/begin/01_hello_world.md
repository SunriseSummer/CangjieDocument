# 01. 初识仓颉 (Hello World)

欢迎来到仓颉编程世界！编程不仅仅是写代码，更是与计算机对话的艺术。让我们从最小可运行的程序开始，建立一个“能跑、可观察、可扩展”的起点。

## 本章目标

*   了解仓颉程序的入口 `main` 及最小可运行结构。
*   熟悉字符串插值与基础输出，形成“编译-运行”的流程习惯。
*   认识源文件命名与输出信息在工程中的作用。

## 1. 环境准备

在开始之前，请确保你已经安装了仓颉编译器 (`cjc`)。

验证安装：
```bash
cjc -v
```

## 2. 你的第一行魔法咒语

创建一个名为 `hello.cj` 的文件。我们不仅要打印问候语，还会输出一段“启动自检”信息，模拟真实项目的启动反馈。

```cangjie
main() {
    println("你好，仓颉！")
    println("启动自检完成，准备就绪。")

    // 简单的计算，展示程序不仅仅是打印文本
    let year = 2024
    println("欢迎来到 ${year} 年的仓颉世界！")
}
```

*   `main()`: 程序的灵魂入口。
*   `println`: 最常用的输出指令。
*   `${year}`: 字符串插值，让变量直接嵌入文本中。

## 3. 编译与施法 (运行)

将代码编译成可执行文件：

```bash
cjc hello.cj -o hello
```

运行它：

```bash
./hello
```

输出：
```text
你好，仓颉！
启动自检完成，准备就绪。
欢迎来到 2024 年的仓颉世界！
```

恭喜！你已经成功唤醒了你的第一个仓颉程序。

## 工程化提示

*   实际项目中建议为输出添加模块前缀或时间戳，方便日志检索。
*   `main` 中只保留启动与调度逻辑，复杂流程应拆分为函数或模块。
*   类似年份/环境参数建议集中配置，避免散落在代码里。
