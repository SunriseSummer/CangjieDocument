# 仓颉语言类与接口 Skill

## 1. 类（class）

### 1.1 定义
- 使用 `class ClassName { ... }` 声明，仅可在顶层作用域定义
- **引用类型** — 赋值/传参共享同一对象

### 1.2 抽象类
- `abstract class` — 可声明抽象函数（无函数体），不能实例化
- 抽象函数不能为 `private`；非抽象子类须实现所有抽象函数
- `sealed` 修饰符限制继承仅在同一包内

### 1.3 成员变量
- **实例成员**：使用 `let`/`var` 定义，通过对象访问。可省略初始值（须有类型标注）
- **静态成员**：`static let`/`static var`，须有初始值（除非有静态初始化器），通过类型名访问

### 1.4 静态初始化器
- `static init() { ... }` — 每个类最多一个。须初始化所有未初始化的静态成员。不允许访问修饰符

### 1.5 构造函数
- **普通构造函数**：`init(params) { ... }` — 须初始化所有未初始化的实例成员。多个构造函数须构成有效重载
- **主构造函数**：`ClassName(let x: T, var y: T) {}` — 参数名前缀 `let`/`var` 同时定义成员变量。每个类最多一个
- **执行顺序**：(1) 构造函数外成员默认值 (2) 隐式/显式 `super()` 调用 (3) 构造函数体
- **自动生成构造函数**：若无自定义构造函数且所有实例成员有初始值，则自动生成无参 `init()`

### 1.6 终结器（`~init`）
- `~init() { ... }` — 在垃圾回收时触发。无参数、无返回类型、无修饰符、不可显式调用
- 有终结器的类不能为 `open`。每个类最多一个。不能在扩展中定义
- 时机不确定；可能在任意线程上运行

### 1.7 成员函数
- **实例函数**：通过对象访问，可使用 `this`
- **静态函数**：`static func`，通过类型名访问，不能访问实例成员
- **抽象函数**：无函数体，仅在抽象类或接口中。隐式 `open`。须为 `public` 或 `protected`

### 1.8 访问修饰符（4 级）
- `private` — 仅在类内可见
- `internal`（默认） — 当前包及子包可见
- `protected` — 当前模块及子类可见
- `public` — 全局可见

### 1.9 `This` 类型
- 在类内部，`This` 是当前类类型的占位符。仅可用作实例方法返回类型
- 子类继承返回 `This` 的方法时，返回类型被识别为子类类型

### 1.10 对象创建
- `let obj = ClassName(args)` — 调用构造函数
- `var` 成员可通过对象修改；`let` 成员不可修改
- 赋值复制**引用**，非对象本身

### 1.11 继承
- 单继承：`class B <: A { }`。`A` 须为 `open` 或 `abstract`
- 子类继承所有非 `private`、非构造函数成员
- `sealed` 抽象类 = 仅同包内可继承。`sealed` 隐含 `public`/`open`
- 类不能继承自身。无显式父类的类继承自 `Object`

### 1.12 父类构造函数调用
- 在子类 `init` 中：使用 `super(args)` 或 `this(args)` 作为函数体**第一个表达式**（互斥）
- 主构造函数可用 `super(args)` 但不可用 `this(args)`
- 若两者都未调用，编译器插入 `super()` — 父类须有无参构造函数

### 1.13 重写与重定义
- **重写（实例函数）**：父类函数须为 `open`；子类使用 `override`（可选）。基于运行时类型的动态分派
- **重定义（静态函数）**：子类使用 `redef`（可选）。基于类型名的静态分派
- 命名参数须在父类与重写之间匹配
- 泛型重写时子类约束须**等于或宽松于**父类

## 2. 接口（interface）

### 2.1 定义
- `interface I { ... }` — 定义抽象类型，无数据。成员：函数、运算符重载、属性
- 成员隐式 `public`（不允许额外访问修饰符）。实现者须使用 `public`
- `interface` 隐式 `open`
- `sealed interface` — 仅同包内可继承/实现/扩展。隐含 `public`/`open`

### 2.2 实现
- `class Foo <: I { ... }` — 须实现 `I` 的所有成员。`Foo` 成为 `I` 的子类型
- 多接口：`class C <: I1 & I2 { ... }`（无顺序要求）
- 可实现接口的类型：除 Tuple、VArray 和函数类型外的所有类型
- 三种实现方式：(1) 类型定义处 (2) 通过扩展 (3) 语言内置

### 2.3 实现规则
- 函数：名称、参数列表、返回类型须相同。例外：若接口返回类型为 `class`，实现者可返回子类
- 属性：`mut` 修饰符须匹配；类型须相同

### 2.4 接口继承
- 接口可继承一个或多个接口（不能继承类）：`interface I3 <: I1 & I2 { ... }`
- 可添加新成员。实现 `I3` 须实现 `I1`、`I2`、`I3` 的所有成员

### 2.5 默认实现
- 接口成员可有默认实现（实例和静态均可）
- 有默认实现的静态成员可通过接口名和实现者名访问
- 无默认实现的静态成员不能通过接口名直接访问
- 若多个继承接口提供同一成员的冲突默认实现，实现者**须**提供自己的实现（菱形继承解决）

### 2.6 泛型成员
- 接口可定义具有 `open` 语义的泛型实例/静态函数

### 2.7 `Any` 类型
- 内置接口：`interface Any {}`
- 所有接口继承 `Any`；所有非接口类型实现 `Any`。每个类型都是 `Any` 的子类型

## 3. 属性（prop）

### 3.1 定义
- `prop name: Type { get() { ... } }` — 只读属性（类似 `let`）
- `mut prop name: Type { get() { ... } set(v) { ... } }` — 读写属性（类似 `var`）
- 可在 `interface`、`class`、`struct`、`enum`、`extend` 中定义

### 3.2 getter 与 setter
- **getter**：`() -> T` — 读取属性时执行
- **setter**：`(T) -> Unit` — 属性被赋值时执行
- 在 getter/setter 内部访问属性本身 = 递归调用

### 3.3 `mut` 限制
- 数值类型、元组、函数、`Bool`、`Unit`、`Nothing`、`String`、`Range`、`enum` 类型**不能**有 `mut` 属性

### 3.4 修饰符
- 与成员函数相同的访问修饰符（`public`、`private`、`protected`、`internal`）
- 支持 `open`、`override`、`redef` — 允许在子类中重写/重定义
- 重写时若父类属性有 `mut`，子类也须有 `mut`，类型相同

### 3.5 抽象属性
- 在 `interface` 和 `abstract class` 中：属性声明无实现体
- 实现者/非抽象子类须提供实现

### 3.6 实例属性与静态属性
- 实例属性：通过对象访问（`obj.prop`）
- 静态属性：通过类型名访问（`ClassName.prop`）。可为 `static mut prop`

## 4. 子类型关系

### 4.1 子类型多态
- 若参数/变量/返回类型为 `T`，可接受 `T` 的任何子类型

### 4.2 子类型关系来源
1. **类继承**：`class Sub <: Super {}` → `Sub <: Super`
2. **接口实现**（包括通过扩展）：→ 类型 <: 接口
3. **元组类型**：`(C2, C4) <: (C1, C3)` 若 `C2 <: C1` 且 `C4 <: C3`（协变，逐元素）
4. **函数类型**：`(U1) -> S2 <: (U2) -> S1` 当且仅当 `U2 <: U1`（参数逆变）且 `S2 <: S1`（返回协变）
5. **恒真关系**：
   - `T <: T`（自反性）
   - `Nothing <: T` 对所有 `T` 成立
   - `T <: Any` 对所有 `T` 成立
   - `C <: Object` 对任何类 `C` 成立
6. **传递性**：若 `A <: B` 且 `B <: C`，则 `A <: C`

## 5. 类型转换

### 5.1 无隐式转换
- 仓颉无隐式类型转换（子类型到父类型不被视为"转换"）

### 5.2 数值转换
- `T(e)` 其中 `T` 和 `e` 为任意数值类型
- 可能溢出：编译时检测 → 错误；运行时 → 异常

### 5.3 `Rune` ↔ 整数转换
- `UInt32(runeExpr)` — 返回 Unicode 标量值
- `Rune(intExpr)` — 仅当值在 `[0x0000, 0xD7FF]` 或 `[0xE000, 0x10FFFF]` 范围内有效

### 5.4 `is` 运算符
- `e is T` → `Bool`。当 `e` 的**运行时类型**是 `T` 的子类型时返回 `true`

### 5.5 `as` 运算符
- `e as T` → `Option<T>`。运行时类型是子类型时返回 `Some(e)`，否则返回 `None`
- 安全向下转型机制 — 无异常，仅返回 `Option`
