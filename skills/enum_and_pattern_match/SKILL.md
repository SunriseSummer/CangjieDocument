# 仓颉语言枚举与模式匹配 Skill

## 1. 枚举类型

### 1.1 定义规则
- 使用 `enum` 关键字 + 名称 + `{}` 体定义
- 体内包含**构造函数**，以 `|` 分隔（第一个 `|` 可选）
- 每个枚举须有**至少一个具名构造函数**
- 构造函数可以是**无参**（`Red`）或**有参**（`Red(UInt8)`）
- **同名构造函数**在参数数量不同时允许（包括 0 个参数）
- **非穷举枚举**：可有一个特殊的未命名 `...` 构造函数作为最后一个。不能直接匹配，须用 `_` 或绑定模式
- **递归定义**支持（构造函数参数可引用枚举自身）
- 枚举体还可包含**成员函数**、**运算符函数**和**成员属性**，但不能与构造函数同名
- **枚举仅能在顶层作用域定义**

### 1.2 使用规则
- 通过 `TypeName.Constructor` 或直接使用构造函数名创建实例
- **名称冲突**：若构造函数名与作用域中的变量、函数或类名冲突，**须**用枚举类型名限定

## 2. match 表达式

### 2.1 两种形式

**形式 1 — 有匹配值：**
```cangjie
match (expr) {
    case pattern1 => expr1
    case pattern2 => expr2_line1
                      expr2_line2
    case _ => defaultExpr
}
```
- `=>` 后可以是**一个或多个表达式、变量定义和函数定义**，多个时各占一行即可，**不需要用 `{}` 包裹**
- 每个 `case` 分支的代码块的值/类型 = 最后一个表达式的值/类型
- 每个 `case` 可用 `|` 连接多个模式（须为同类模式）
- `=>` 后新定义的变量/函数的作用域从其定义处到下一个 `case` 之前结束

**形式 2 — 无匹配值：**
```cangjie
match {
    case boolExpr1 => expr1
    case boolExpr2 => expr2_line1
                      expr2_line2
    case _ => defaultExpr   // _ 表示 true
}
```
- 每个 `case` 接受 `Bool` 表达式（非模式）
- `=>` 后同样可以是一个或多个表达式/定义，各占一行，**不需要 `{}`**
- 此形式不支持模式守卫

### 2.2 穷举性
- **所有 `match` 表达式须穷举** — 须覆盖所有可能值。非穷举 → **编译错误**
- 常见做法：使用通配符 `_` 作为最后一个 case
- **非穷举枚举**（`...` 构造函数）须使用 `_` 或绑定模式覆盖

### 2.3 模式守卫（`where`）
- 模式后可添加 `where condition`（条件须为 `Bool` 类型）
- case 仅在模式匹配**且**守卫为 true 时才匹配

### 2.4 执行语义
- case 按**从上到下**顺序求值，首个匹配执行后退出 match
- **无穿透**（fall-through）

### 2.5 match 表达式类型
- **有显式上下文类型**：每个分支体须为期望类型的子类型
- **无上下文类型**：match 表达式类型为所有分支体的**最小公共父类型**
- **值未使用时**：类型为 `Unit`，无公共父类型要求

## 3. Option 类型

### 3.1 定义
```cangjie
enum Option<T> {
    | Some(T)
    | None
}
```
- `Some(v)` = 有值；`None` = 无值

### 3.2 简写语法
- `?Ty` 等价于 `Option<Ty>`

### 3.3 自动包装
- 当上下文期望 `Option<T>` 时，可直接传 `T` 值 — 编译器自动包装为 `Some`
```cangjie
let a: ?Int64 = 100      // 自动包装为 Some(100)
```

### 3.4 显式 `None` 带类型
- 无上下文类型时使用 `None<T>` 语法

## 4. 模式类型

### 4.1 常量模式
- 整数、浮点、字符、布尔、字符串字面量（无插值）和 `Unit` 字面量
- 须与匹配目标**同类型**
- 值相等时匹配
- 多个常量可用 `|` 组合
- 匹配 `Rune` 值时，`Rune` 字面量和单字符字符串字面量均可作为常量模式
- 匹配 `Byte` 值时，ASCII 字符字符串字面量可作为常量模式

### 4.2 通配符模式（`_`）
- 匹配**任意值**
- 通常用作最后一个 case 捕获剩余情况

### 4.3 绑定模式（`id`）
- 匹配任意值并将其**绑定**到标识符 `id`
- `id` 是**不可变**变量，作用域从引入处到该 `case` 分支结束
- `=>` 后不能修改 `id`
- **不能与 `|`** 连接多个模式一起使用
- **注意**：若 `id` 是枚举构造函数名，则被视为**枚举模式**而非绑定模式

### 4.4 元组模式
- 语法：`(p_1, p_2, ..., p_n)`，每个 `p_i` 是一个模式，`n ≥ 2`
- 每个位置都匹配时整体匹配
- 同一元组模式内不允许重复绑定名
```cangjie
case ("Alice", age) => "Alice is ${age} years old"
```

### 4.5 类型模式
- 两种形式：`_: Type`（无绑定）和 `id: Type`（有绑定）
- 检查值的**运行时类型**是否为 `Type` 的**子类型**
- 若匹配，值被转换为 `Type` 并可选绑定到 `id`

### 4.6 枚举模式
- 镜像枚举构造函数语法：`C`（无参）或 `C(p_1, ..., p_n)`（有参）
- 类型前缀可省略
- 构造函数名匹配**且**所有嵌套模式匹配时整体匹配
- 须**覆盖枚举的所有构造函数**（或使用 `_`）
- **不能在含绑定变量的枚举模式中使用 `|`**

### 4.7 模式嵌套
- 元组和枚举模式可**任意深度嵌套**其他模式

## 5. 模式可反驳性

| 模式 | 可反驳性 | 规则 |
|------|----------|------|
| **常量** | 始终**可反驳** | 可能不匹配值 |
| **通配符**（`_`） | 始终**不可反驳** | 总是匹配 |
| **绑定**（`id`） | 始终**不可反驳** | 总是匹配 |
| **类型**（`id: Type`） | 始终**可反驳** | 运行时类型可能不是子类型 |
| **元组** `(p1,...,pn)` | 当且仅当**所有** `pi` 不可反驳时**不可反驳** | 一个可反驳子模式使整体可反驳 |
| **枚举** `C(p1,...,pn)` | 当且仅当枚举**仅有一个**有参构造函数**且**所有 `pi` 不可反驳时**不可反驳** | 多个构造函数 → 始终可反驳 |

---

## 6. 模式的其他用途

### 6.1 变量定义（let）
- 仅允许**不可反驳**模式
```cangjie
let _ = 100                      // 通配符
let x = 100                      // 绑定
let (x, y) = (100, 200)          // 不可反驳元组（解构）
```

### 6.2 `for in` 表达式
- 仅允许**不可反驳**模式
```cangjie
for (_ in 1..5) { ... }                     // 通配符
for (i in 1..5) { ... }                     // 绑定
for ((i, j) in [(1,2),(3,4),(5,6)]) { ... } // 元组解构
```

### 6.3 `if` 和 `while` 条件
- 可使用 `let pattern <- expression` 条件

### 6.4 这些上下文中允许的不可反驳模式
1. 通配符模式 `_`
2. 绑定模式 `id`
3. 不可反驳的元组模式
4. 不可反驳的枚举模式（单构造函数枚举且所有嵌套模式不可反驳）
