# 标识符

在仓颉编程语言中，标识符（Identifier）是用于给变量、函数、类、接口等程序元素命名的名称。

## 命名规则

仓颉语言支持 Unicode 字符集（标准 15.0.0），这意味着您可以使用中文、英文、数字等多种字符来命名。

标识符分为**普通标识符**和**原始标识符**两类。

### 1. 普通标识符

普通标识符是最常用的命名形式。它必须遵守以下规则：

- **起始字符**: 必须是 `XID_Start` 字符（如英文字母、汉字）或下划线 `_`。
  - **注意**: 数字不能作为起始字符。
  - **注意**: 单独的一个下划线 `_` 不是合法的普通标识符。
- **后续字符**: 可以是 `XID_Continue` 字符（如英文字母、汉字、数字、下划线）。
- **关键字限制**: 不能与仓颉语言的[关键字](../Appendix/keyword.md)完全相同（如 `if`, `while`, `func` 等）。

#### 示例

| 标识符 | 合法性 | 说明 |
| :--- | :--- | :--- |
| `abc` | ✅ 合法 | 标准的英文标识符 |
| `_abc` | ✅ 合法 | 下划线开头 |
| `abc_` | ✅ 合法 | 下划线结尾 |
| `a1b2` | ✅ 合法 | 字母开头，包含数字 |
| `仓颉` | ✅ 合法 | 支持中文 |
| `__こんにちは` | ✅ 合法 | 支持多语言混合 |
| `3abc` | ❌ 非法 | 不能以数字开头 |
| `ab&c` | ❌ 非法 | `&` 不是合法的标识符字符 |
| `_` | ❌ 非法 | 单独的 `_` 不合法（但在模式匹配中有特殊含义） |
| `while` | ❌ 非法 | 关键字不能作为普通标识符 |

### 2. 原始标识符

当您确实需要使用仓颉关键字作为标识符（例如对接外部系统时），可以使用**原始标识符**。

- **语法**: 在标识符首尾加上反引号 `` ` ``。
- **规则**: 反引号内部必须是一个合法的普通标识符或关键字。

#### 示例

| 标识符 | 合法性 | 说明 |
| :--- | :--- | :--- |
| `` `if` `` | ✅ 合法 | 将关键字 `if` 用作标识符 |
| `` `while` `` | ✅ 合法 | 将关键字 `while` 用作标识符 |
| `` `abc` `` | ✅ 合法 | 等价于普通标识符 `abc` |
| `` `3abc` `` | ❌ 非法 | 反引号内必须符合普通标识符规则（数字不能开头） |

## 大小写敏感与规范化

- **大小写敏感**: `Abc` 和 `abc` 是两个不同的标识符。
- **Unicode 规范化**: 仓颉会将所有标识符按照 [NFC (Normalization Form C)](https://www.unicode.org/reports/tr15/tr15-53.html) 形式进行规范化。如果两个标识符在规范化后字符序列相同，则视为同一个标识符。
