# 浮点类型

仓颉语言提供了三种精度的浮点类型，用于表示带有小数部分的数值。它们遵循 IEEE 754 标准。

## 1. 类型概览

| 类型 | 精度标准 | 有效位数 (约) | 说明 |
| :--- | :--- | :--- | :--- |
| **`Float16`** | 半精度 (binary16) | 3 位 | 精度较低，常用于 AI 或图形计算等特定场景 |
| **`Float32`** | 单精度 (binary32) | 6 位 | 常见的浮点类型 |
| **`Float64`** | 双精度 (binary64) | 15 位 | **默认推荐**。精度高，计算误差小 |

> **💡 提示**: 在大多数科学计算和通用场景中，建议优先使用 **`Float64`** 以避免精度丢失问题。

## 2. 浮点类型字面量

### 表示形式
浮点数支持**十进制**和**十六进制**两种写法。

1.  **十进制**:
    - 必须包含小数点 `.` 或指数部分 (`e`/`E`)。
    - 示例: `3.14`, `0.5`, `2e3` (2000.0)

2.  **十六进制**:
    - 前缀为 `0x` 或 `0X`。
    - 必须包含指数部分 (`p`/`P`，底数为 2)。
    - 示例: `0x1.1p0` (1.0625)

### 类型后缀
默认情况下，浮点字面量（如 `3.14`）被推断为 `Float64`。可以使用后缀显式指定类型。

| 后缀 | 类型 | 示例 |
| :--- | :--- | :--- |
| `f16` | `Float16` | `3.14f16` |
| `f32` | `Float32` | `3.14f32`, `2e3f32` |
| `f64` | `Float64` | `0.1f64` |

<!-- compile -->
```cangjie
let pi = 3.14159      // Float64 (默认)
let height = 1.75f32  // Float32
let scientific = 1e-5 // Float64
```

## 3. 支持的操作

浮点类型支持以下操作：
- **算术**: `+`, `-`, `*`, `/`, `%`, `**`
- **比较**: `==`, `!=`, `<`, `>`, `<=`, `>=`
- **复合赋值**: `+=`, `-=`, `*=`, `/=` 等

> **⚠️ 注意**: 浮点类型不支持自增 (`++`) 和自减 (`--`) 操作符。
