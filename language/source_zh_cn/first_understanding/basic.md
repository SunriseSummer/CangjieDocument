# 初识仓颉语言

**仓颉编程语言**是一款面向全场景应用开发的通用编程语言，致力于在开发效率和运行性能之间找到完美的平衡，为开发者提供卓越的编程体验。

仓颉语言的设计不仅追求语法的简洁与优雅，更在核心层面融合了现代编程语言的诸多先进特性。无论是构建高性能的底层系统，还是开发复杂的应用软件，仓颉都能提供强有力的支持。

以下是仓颉语言的主要特点：

- **高效的并发模型**
  仓颉原生支持用户态轻量化线程（协程），并提供了简单易用的并发编程机制。这使得开发者能够轻松编写出高效、高并发的程序，充分利用现代多核处理器的性能。

- **极致的类型与内存安全**
  作为一门静态强类型语言，仓颉通过严格的编译时类型检查，将大量错误拦截在运行之前。同时，自动内存管理机制以及运行时的数组越界、溢出检查等功能，彻底消除了常见的内存安全隐患，让代码更加稳健可靠。

- **简明高效的语法**
  仓颉提供了一系列现代化的语法特性，如插值字符串、主构造函数、Flow 表达式、模式匹配（match）等。这些特性旨在减少样板代码，让开发者能够用更少的代码表达更复杂的逻辑，从而显著提升开发效率。

- **多范式编程支持**
  仓颉完美融合了函数式、命令式和面向对象编程。它不仅拥有高阶函数、代数数据类型、泛型等函数式特性，还具备封装、接口、继承等面向对象特性。开发者可以根据具体场景，灵活选择最适合的编程范式。

- **强大的互操作性与扩展性**
  仓颉支持与 C 语言等现有生态的高效互操作，能够轻松复用成熟的类库。同时，基于词法宏的元编程能力以及尾随 Lambda 等特性，使得仓颉非常适合构建内嵌式领域专用语言（EDSL），满足特定领域的定制化需求。

- **原生支持 UI 开发**
  UI 开发是现代应用的重要组成部分。仓颉通过元编程和独特的语法设计，为构建声明式 UI 框架提供了天然支持，大幅提升了 UI 开发的效率和体验。

- **全场景多后端**
  仓颉支持 CJNative 和 CJVM 两种后端。CJNative 将代码编译为原生二进制，直接运行于操作系统之上，追求极致性能；CJVM 则编译为字节码，运行于虚拟机之上。本文档重点介绍适配 CJNative 后端的内容。

- **功能丰富的标准库**
  仓颉内置了涵盖数据结构、算法、网络通信、文件操作、系统交互等领域的丰富标准库，功能强大且易于使用，做到真正的“开箱即用”。
